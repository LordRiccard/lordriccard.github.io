Projeto Linkage

União dos bancos do DATASUS do SIM e SINASC em um único arquivo. A imagem mostra meu mapa mental feito no aplicativo ObsidianMD.

#Python #Pandas #RecordLinkage #JupyterNotebook

GitHub
Esta é apenas uma parte do que já foi desenvolvido para o Projeto de Pesquisa. O objetivo primário dessa parte foi efetuar o linkage dos bancos, onde um único arquivo seria criado contendo apenas registros que pertencem a uma única pessoa presentes em ambos os bancos ao mesmo tempo. O objetivo secundário foi fornecer um conjunto de arquivos otimizado para cada banco, contendo os arquivos referentes a cada ano individualmente e um arquivo com todos os anos, possibilitando o uso em pesquisas futuras.
Esse projeto me ajudou muito a entender muitos sobre o pandas, em específico, como o mesmo gere tipos de dados, para otimização do uso da memória.

Em Abril desse ano, fui escolhido para participar de um projeto de pesquisa, cujo foco era a análise de dados voltado para a saúde gestacional de grupos de risco. Este projeto me proporcionou muito conhecimento sobre a área de ciência de dados como um todo, por meio das implementações e análises que fiz, e o verdadeiro impacto que a análise de dados pode trazer para o mundo. Uma dessas implementações, chamada Projeto Linkage, pode ser vista na seção de Vitrine de Projetos.
Os dados usados trouxeram alguns desafios muito importantes para o meu início na área, a exemplo de seu tamanho máximo, mais de 110 GB de tamanho em disco, e do tempo necessário Fpara processamento, mais de 4 horas para todos os arquivos.

Esses desafios me ajudaram a entender um pouco sobre a realidade do Big Data, que enfrentamos atualmente, e como é importante o desenvolvimento de estratégias para a manipulação efetiva desses dados, que deve abranger desde algo tão simples como a extensão dos arquivos, até quais bibliotecas usar.
